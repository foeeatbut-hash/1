# Лицензирование Flux — инструкция для владельца

Программа при запуске требует активацию: после заставки открывается окно
лицензии, и только после ввода ключа появляется обычный вход по логину/паролю.
Лицензия привязана к одному компьютеру и действует до даты, которую вы задаёте.

## Что у вас есть

- **`license-generator.html`** — офлайн-генератор ключей. Открывается в браузере
  без интернета. **В нём хранится секретный ключ подписи — держите файл в тайне,
  не выкладывайте в сеть и не передавайте никому.** Тот, у кого есть этот файл,
  сможет выпускать лицензии.

## Как выдать лицензию пользователю

1. Пользователь запускает программу. В окне лицензии он видит **«Код этого
   компьютера»** (например `A1B2-C3D4-E5F6-7890`) и присылает его вам.
2. Вы открываете `license-generator.html`, вводите этот код, выбираете дату
   окончания (или кнопку +30/+60/+90 дней) и нажимаете «Сгенерировать».
3. Копируете полученный ключ (`FLUX1.…`) и отдаёте пользователю.
4. Пользователь вставляет ключ в окно лицензии и нажимает «Активировать» →
   открывается вход.

## Продление / новый компьютер

- **Продлить**: сгенерируйте новый ключ для того же кода компьютера с новой датой.
- **Другой компьютер**: у него будет другой код — сгенерируйте ключ под него.
  Один ключ работает только на том компьютере, для которого выпущен.

## Почему это нельзя обойти

- Ключи подписаны криптографически (Ed25519). В программе есть только ключ
  **проверки**; ключ **подписи** — только в вашем генераторе. Поэтому подделать
  лицензию или «выдать доступ через базу данных» нельзя даже с полным доступом к
  базе, настройкам и исходникам — секретного ключа там нет.
- Проверка идёт на самой машине пользователя, срок — по защищённому времени
  (перевод часов назад не продлевает лицензию).
- Привязка к компьютеру — по устойчивому отпечатку оборудования.

Честное ограничение: ни одна программа, работающая на чужом компьютере, не
защищена на 100% от вскрытия самого exe опытным взломщиком. Но выпускать
действующие ключи или обойти лицензию через базу/настройки невозможно —
а это и есть защита от несанкционированной выдачи доступа.
