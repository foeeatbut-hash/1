# Настройка сервера MariaDB 10 для Flux — инструкция для программиста

Задача: поднять сервер MariaDB, создать пустую базу с таблицами, открыть сетевой
доступ. **Пароли пользователей приложения и учётные записи Flux вас не касаются —
их задаёт администратор в самой программе.** Вы отдаёте только адрес сервера, имя
базы, имя пользователя базы и временный пароль базы; администратор его сразу сменит.

## 1. Установка (Windows Server)

1. Скачать MariaDB 10.11 LTS: https://mariadb.org/download/ → MSI-установщик.
2. При установке:
   - кодировка по умолчанию — **UTF8MB4** (галочка «Use UTF8 as default»);
   - задать пароль root (останется только у вас на время настройки);
   - службу оставить включённой (порт 3306).

## 2. Создать базу и пользователя приложения

Открыть «MySQL Client (MariaDB 10.11)» под root и выполнить. Замените
`ВРЕМЕННЫЙ_ПАРОЛЬ` на любой временный — администратор его потом сменит:

```sql
CREATE DATABASE flux CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'flux'@'%' IDENTIFIED BY 'ВРЕМЕННЫЙ_ПАРОЛЬ';
GRANT ALL PRIVILEGES ON flux.* TO 'flux'@'%';
FLUSH PRIVILEGES;
```

Пользователь `flux` имеет права только на базу `flux` — не на весь сервер.

## 3. Создать таблицы

Положить рядом файл `flux-mariadb-init.sql` и выполнить в командной строке:

```
mysql -u flux -p flux < flux-mariadb-init.sql
```

(введёте ВРЕМЕННЫЙ_ПАРОЛЬ). Скрипт создаёт 31 таблицу и ничего больше — ни
пользователей, ни паролей приложения внутри нет. Проверка:

```
mysql -u flux -p flux -e "SHOW TABLES;"
```

Должны появиться таблицы User, Project, DocRegister, FileNode, ChatMessage и т.д.

## 4. Открыть доступ по сети

1. В `my.ini` (обычно `C:\Program Files\MariaDB 10.11\data\my.ini`), секция
   `[mysqld]`:
   ```
   bind-address = 0.0.0.0
   ```
2. Перезапустить службу MariaDB (services.msc → MariaDB → Перезапустить).
3. В брандмауэре Windows разрешить входящие TCP на порт **3306** для локальной сети.

## 5. Что передать администратору

- IP-адрес сервера и порт (3306);
- имя базы: `flux`;
- имя пользователя базы: `flux`;
- **ВРЕМЕННЫЙ_ПАРОЛЬ** (одноразово — админ его сменит по своей инструкции).

На этом ваша часть закончена. Учётные записи сотрудников и пароль главного
администратора создаются внутри программы при первом подключении — доступа к ним
у вас нет и не требуется.
