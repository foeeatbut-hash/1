# Подключение Flux к MariaDB — инструкция для администратора

Программист поднял сервер и передал вам: IP-адрес, имя базы `flux`, пользователя
базы `flux` и **временный пароль базы**. Дальше — ваша часть. Смысл: после смены
пароля базы программист теряет к ней доступ, а учётные записи Flux (включая ваш
пароль главного администратора) он не видел вообще — они живут внутри программы.

## 1. Сменить пароль базы (чтобы у программиста не осталось доступа)

На сервере откройте «MySQL Client» под root (или попросите программиста дать вам
одноразовый доступ root на эту одну команду) и выполните, придумав свой пароль:

```sql
ALTER USER 'flux'@'%' IDENTIFIED BY 'ВАШ_НОВЫЙ_ПАРОЛЬ';
FLUSH PRIVILEGES;
```

Теперь пароль базы знаете только вы. Временный пароль от программиста больше не
работает.

## 2. Подключить программу

На каждом рабочем месте: **Настройки → База данных → «Совместный / Внешний»**,
строка подключения:

```
mysql://flux:ВАШ_НОВЫЙ_ПАРОЛЬ@IP-СЕРВЕРА:3306/flux
```

Нажать «Тестировать» → «Сохранить и подключить». Программа перезапустится в
совместном режиме.

## 3. Первый вход и смена пароля администратора

При первом подключении к пустой базе программа сама создаёт главного
администратора:

- логин: `RaupovKhKh`
- пароль: `1122`

**Сразу войдите и смените пароль** (Настройки → профиль). Пароли в базе хранятся
в виде необратимого хэша — даже тот, у кого есть доступ к базе, ваш пароль не
увидит.

## 4. Подключить остальных пользователей

Дайте коллегам ту же строку подключения (`mysql://flux:…@IP:3306/flux`) — они
вводят её один раз в Настройках. Дальше все работают с одной общей базой: ВДР,
документы, файлы Проводника, теги, чат и уведомления — общие для всех.

---

**Кто что знает в итоге:**

| Секрет | Программист | Вы (админ) |
|---|---|---|
| Пароль базы `flux` | нет (сменили) | да |
| Пароль root сервера | да (на время настройки) | по желанию |
| Пароль админа Flux (`RaupovKhKh`) | нет | да |
| Пароли сотрудников | нет (хэши в базе) | нет (только сброс) |
