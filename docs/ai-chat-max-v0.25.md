# ИИ-чат: максимальная интеграция + WOW (проект, без применения)

Опирается на то, что уже есть в коде: bulk-эндпоинт `PUT /api/tags/bulk-metadata`,
система подсветки шагов (`AssistantSpotlight` — highlight+advance), deep-links
(`/registry?focus=/dup=`, `/notes?new=`), знание текущего раздела (`setRoute`).

---

## 1. Недочёты и упрощения (убрать/почистить)

Найдено в коде:
- **Мёртвый код** в `assistantStore.ts`: функции `tokensFrom` и `fieldMatches` больше
  не используются (заменены на `nlp.ts`), плюс дублируется `STOPWORDS` (есть в `nlp.ts`).
  → удалить ~30 строк, один источник правды.
- **Тумблер «Демонстрация»** теперь избыточен: «как …» и так авто-распознаётся и даёт
  кнопку тура. Тумблер занимает целую строку панели. → убрать переключатель, а обучающий
  режим включать автоматически (по «научи», «покажи по шагам»). Панель станет чище.
- **Устаревание данных**: кэш ассистента (15 с) не сбрасывается после изменений в
  программе — «сколько тегов» может врать 15 с, а после действий самого чата (создал
  заметку, сменил этап) данные не обновляются. → `bumpAssistantData()` на мутациях + сброс
  после действий чата. Ответы всегда свежие.
- **«Показать в Менеджменте»** ведёт в раздел, но не подсвечивает конкретную позицию.
  → deep-link `/management?focus=<id>` с прокруткой и подсветкой строки (как уже сделано
  для дерева/спецификации).

Итог: минус тумблер и ~30 строк мёртвого кода, данные всегда актуальны — меньше, проще, точнее.

## 2. Максимальная интеграция: чат не только показывает, а ДЕЛАЕТ

Сейчас действия чата — в основном навигация. Чтобы «прям интегрировать», добавить
операции прямо из чата, безопасно (превью → подтверждение → выполнение → отмена):

### 2.1 Массовые операции на естественном языке ⚡ (главное)
- «переведи все клапаны в “заказан”», «пометь куплено вентиляторы вытяжки»,
  «спиши дубли клапанов» → чат показывает **что изменит** («Затрону 8 позиций: …»),
  ты жмёшь «Выполнить», он делает одним запросом (`bulk-metadata`) и предлагает **Отмену**.
- Это ускоряет рутину закупок в разы и невозможно ни в одном мессенджере.

### 2.2 Создание из чата
- «добавь тег 3700-K05-HV-101 в отдел ОВиК» → создаёт тег и открывает его на холсте.
- «создай заметку про подбор калорифера» → уже есть; расширить: сразу вписать шаблон.

### 2.3 Действия по текущему разделу (контекст экрана)
Чат знает, где ты (`currentRoute`). Глаголы действуют на активный экран:
- на **холсте**: «выдели вентиляторы» → выделяет карточки; «упорядочь» → авто-раскладка.
- в **Спецификации/Менеджменте**: «покажи только критичные» → применяет фильтр экрана.
- в **Оборудовании**: «разверни всё» / «покажи конфликты».

### 2.4 «Спросить ИИ о выделенном»
Выделил карточки на холсте или строки в спецификации → ПКМ «Спросить ИИ» → чат берёт
их в контекст: «Эти 5 позиций: 2 критичны, 3 не заказаны — показать/выгрузить/сменить этап».

## 3. Три WOW-функции

### WOW-1. Ctrl+K — универсальная командная строка ⌘
Превратить помощника (уже открывается по Ctrl+K) в **командный центр**, как в Linear/
Superhuman: одно поле, куда пишешь что угодно, и оно само решает —
перейти в раздел, найти тег/позицию, выполнить действие или ответить. Мгновенно,
с клавиатуры, без мыши. Список результатов со стрелками ↑↓ и Enter.
- «мен» → перейти в Менеджмент. «3700-K02» → карточка. «дубли» → список.
  «заказать клапаны» → массовое действие. Один вход во всю программу.

### WOW-2. Массовые действия на языке с превью и Undo 🪄
(см. 2.1) — самое «интегрированное» и самое полезное для менеджера. Ключевое:
безопасность (превью + подтверждение + откат), поэтому не страшно доверить чату правки.

### WOW-3. Утренний дайджест проекта 📋 (проактивность)
При входе — короткая сводка «что произошло и что горит», собранная из логов и
показателей: «Со вчера: +12 тегов, 3 стали критичными, появился 1 дубль, 5 позиций
ждут заказа». Каждая строка — кнопка «Показать». Менеджер за 5 секунд видит состояние
проекта, не открывая ничего. Показывать раз в день/сессию, ненавязчиво, с «Скрыть».

Доп. идея (лёгкая): **智能-подсказки по состоянию** — если открыл раздел с проблемами,
чат сам предлагает «Тут 4 дубля — показать?» (данные уже собираются).

## 4. Что это даёт и что убираем

Добавляем: массовые операции языком (превью+undo), Ctrl+K командную строку, дайджест,
действия по контексту экрана, «спросить о выделенном».
Убираем/упрощаем: тумблер «Демонстрация», мёртвый код, устаревание данных.
Не добавляем: голос (цех), внешние LLM/интернет (офлайн и приватность).

## 5. Порядок (когда скажешь)
1. Чистка: мёртвый код, свежесть данных, deep-link в Менеджмент (быстро, основа).
2. WOW-2: массовые операции языком с превью+undo (максимальная польза).
3. WOW-1: Ctrl+K командная строка (максимальное удобство).
4. WOW-3: дайджест + подсказки по состоянию.
5. Контекстные действия по экрану и «спросить о выделенном».
