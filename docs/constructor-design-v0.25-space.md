# Конструктор, часть III: пространство раздела, жизненный цикл файла, Проводник

Продолжение частей I (обзор) и II (подсистемы). Здесь — как раздел выглядит при входе,
как файл живёт от создания до закрытия, «мои/общие», просмотр/редактирование и
автоматическое присутствие в Проводнике. Мерило каждого решения — **минимум действий**:
для каждого сценария ниже посчитан путь в кликах (§9).

---

## 1. Вход в раздел: один экран, всё под рукой

Никаких промежуточных «выберите пространство». Один экран Библиотеки:

```
┌────────────────────────────────────────────────────────────────────┐
│  Конструктор                     [Поиск…]   [+ Новый] [Собрать данные]
│
│  ► Продолжить                                    ← только если есть
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│  │ Габариты AHU │ │ Ведомость…  │ │ Без названия │   мои последние
│  │ вчера, 18:40 │ │ 3 дня назад │ │ ЧЕРНОВИК     │   1 клик = редактирование
│  └─────────────┘ └─────────────┘ └─────────────┘
│
│  Мои файлы (12)          Общие файлы (34)         Шаблоны (7)
│  ─────────────           ────────────────         ──────────
│  список/карточки с сортировкой и фильтрами; ниже — свёрнутая Корзина
└────────────────────────────────────────────────────────────────────┘
```

- **«Продолжить»** — до 3 моих последних редактированных документов. Клик открывает
  **сразу в редактировании** (это единственное место с таким поведением — понятно
  почему: «продолжить» и значит продолжить). Свою незаконченную работу человек
  возвращает за 1 клик от входа в раздел.
- **Мои файлы** — мои личные (`scope=PERSONAL`, виден только мне) **и** общие, где я
  автор. Т.е. «моё» — по авторству, а не только по приватности: свою работу человек
  ищет в одном месте, не вспоминая, что там у неё за scope. У личных — значок замка.
- **Общие файлы** — все `SHARED` проекта, группировка/сортировка: по дате изменения
  (умолчание), по автору, по имени. Фильтр «только с моим участием».
- **Шаблоны** — из части I (общие + мои), с кнопкой «Создать по шаблону» прямо на карточке.
- **Поиск** — по имени и автору, мгновенный, ищет сразу во всех трёх группах.
- Карточка: имя, автор, дата, значки состояния: 🔒 личный, ✏️ «сейчас редактирует
  Иванов» (live через socket), ⚠️ черновик без имени, 🟡 stale («данные проекта
  изменились» — из §II.2.4, видно ещё ДО открытия), 📌 закреплён.
- **Закрепить** (pin) — закреплённые всплывают наверх своей группы; личная настройка.
- **Корзина** — свёрнутая секция внизу: мягкое удаление, хранится 30 дней
  (`deletedAt` в `ConstructorDoc`), восстановление в 1 клик. Из корзины истёкшее
  чистит фоновая задача. Личные удаляет только владелец; общие — автор или право
  `constructor.manage` (менеджер/админ).

## 2. Просмотр по умолчанию, редактирование по переключателю

Клик по любому файлу (кроме «Продолжить») → **режим просмотра**. Это решает сразу три
задачи: никто случайно не портит чужой документ; блокировка (§II.10) не захватывается
зря — десять человек могут смотреть одновременно; открытие мгновенное (просмотру не
нужны каталог полей и кэш формульных данных — только снапшот).

В просмотре доступно всё «читательское»: скролл, выделение, копирование, фильтры-на-лету
(не сохраняются), экспорт/печать, версии, клик-переходы к тегам. Сверху — тонкая
панель: `👁 Просмотр  [✏️ Редактировать]  Иванов сейчас редактирует…`

Переключение в редактирование — **одна кнопка** (или клавиша `E`, или двойной клик по
любой ячейке — три привычки, один результат):

- документ свободен → мгновенно берётся блокировка, панель инструментов «выезжает»,
  режим редактирования;
- редактирует другой → кнопка показывает «Редактирует Иванов» с действиями
  «Попросить» (уведомление Иванову через существующие Notifications: «Петров ждёт
  документ …» — Иванов закрывает, Петрову приходит «документ освободился, [Открыть]»)
  и «Перехватить» (протокол §II.10);
- нет права `constructor.edit` на этот документ (чужой личный не открывается вовсе —
  §4) → кнопки просто нет.

Обратно `Редактирование → Просмотр` — той же кнопкой: автосейв дожимается, блокировка
отпускается. Уход с экрана/закрытие вкладки делает то же автоматически — «забыл выйти
из редактирования» не держит файл заблокированным дольше TTL (45 с, §II.10).

## 3. Жизненный цикл: от «+ Новый» до закрытия

### 3.1. Создание — ноль вопросов на старте

`+ Новый`, «Собрать данные» или «По шаблону» → документ создаётся **сразу**, без
единого диалога: имя `Без названия — 10 июля`, `scope=SHARED` (умолчание по ТЗ),
зеркало в системной папке Проводника (§5), режим редактирования, блокировка взята.
Человек думает о таблице, а не о бюрократии. Всё, что мы «не спросили», дособерём при
закрытии — один раз и одним диалогом.

### 3.2. Во время редактирования

Шапка редактора — все «свойства файла» в одну строку, каждое меняется по месту без
диалогов:

```
[← Закрыть]  Габариты AHU ✎   |  Общий ▾  |  📁 Конструктор/ ▾  |  ⟳ автосохранено 14:02
```

- **Имя** — кликабельно, правится инлайн в любой момент (тогда диалог при закрытии
  вообще не появится).
- **Общий/Личный** — переключатель-селект. Общий→Личный: документ исчезает у остальных
  из Библиотеки и Проводника; если его кто-то сейчас смотрит — у него баннер «автор
  сделал документ личным», просмотр закрывается. Переключать может **только автор**
  (и админ). Личный→Общий — просто публикация, доступна владельцу всегда. Оба перехода
  пишутся в `SystemChangeLog`. Зеркало в Проводнике автоматически переезжает между
  общим и личным разделами (§5.3) — руками ничего делать не надо.
- **Папка** — текущее расположение зеркала в Проводнике; клик → мини-дерево папок
  Проводника, выбор — зеркало переехало. По умолчанию — системная папка, трогать
  не обязательно никогда.
- **Автосейв-индикатор** — как в карточке тега: «сохраняю…/сохранено чч:мм»; никакой
  кнопки «Сохранить» нет в принципе (Ctrl+S на всякий случай форсирует автосейв и
  мигает индикатором — привычке больно не делаем).

### 3.3. Закрытие — единственный диалог, и только когда он нужен

Кнопка `← Закрыть` (или Esc в режиме просмотра, или переход в другой раздел):

- имя уже человеческое (переименовывали хоть раз) → **никакого диалога**, просто
  закрылись в Библиотеку;
- имя автогенерированное → компактный диалог:

```
┌ Как назвать документ? ──────────────────────────┐
│  [Габариты и веса — AHU_______________]  ← фокус, текст выделен
│   предложения: «Габариты и веса — AHU» · «Теги: AHU, вентиляторы»
│   Общий · 📁 Конструктор/        ← текущие, кликабельны для смены
│  [Оставить черновиком]                 [Готово ⏎] │
└──────────────────────────────────────────────────┘
```

- **Умное предложение имени** — из содержимого: имя первого блока или «сущность +
  главный фильтр» его запроса («Габариты и веса — AHU»), при пустом документе —
  «Таблица — 10 июля». Предложение уже вписано и выделено: согласен — просто Enter
  (итого закрытие = 2 нажатия), не согласен — печатаешь поверх.
- **«Оставить черновиком»** — законная кнопка, не наказываем: документ остаётся с
  авточерновым именем и бейджем ЧЕРНОВИК в «Моих», зеркало в Проводнике у черновика
  **не создаётся** (мусор из недоделок не сыплется коллегам в папку; личное же
  пространство — «Продолжить» и «Мои» — черновик показывает всегда).
- Крах/выключение до именования = «Оставить черновиком» автоматически (документ-то
  сохранён автосейвом). Восстановление несохранённого — §II.7.3.
- Уникальность имени в проекте: тихо дописываем « (2)» — диалогом не пристаём.

### 3.4. Правило одного диалога

Инвариант UX всего раздела: **на жизненный цикл файла — максимум один системный
диалог** (именование при закрытии, и то пропускаемый). Всё остальное — переключатели
по месту, автосейв, умолчания. Любая новая фича раздела проверяется этим правилом.

## 4. Видимость и права (сводка)

| Действие | Личный | Общий |
|---|---|---|
| Видеть в Библиотеке/Проводнике | только владелец (админ не видит — как личные папки Проводника) | все с `constructor.view` |
| Открыть просмотр | владелец | все с `constructor.view` |
| Редактировать | владелец | все с `constructor.edit` (блокировка — §II.10) |
| Переключить Личный/Общий | владелец | только автор (+админ) |
| Переместить зеркало в папку | владелец | любой редактирующий |
| Удалить (в корзину) | владелец | автор или `constructor.manage` |
| Экспортировать | владелец | все с `constructor.export` |

«Автор» = `createdById`. Смена автора не предусмотрена (дублирование даёт новый
документ с новым автором — этого достаточно).

## 5. Зеркало в Проводнике

### 5.1. Системная папка

При первом документе проекта в Проводнике (общий раздел) создаётся папка
**«Конструктор»**: `Folder.system = true` (новое поле) — сервер отклоняет её удаление,
переименование и перенос; в UI Проводника у неё значок раздела и нет пунктов
удалить/переименовать. Аналогичная папка в личном разделе пользователя — для личных
документов (создаётся лениво). Внутри системной папки пользователи могут заводить
подпапки как угодно — защищена только она сама.

### 5.2. Что за файл лежит в Проводнике

Это **живое зеркало, не копия**: `FileNode.type = "CONSTRUCTOR"`, `refId = docId`
(новое поле), без физического файла на диске. Ведёт себя как родной файл Проводника:

- имя = имя документа (переименование с любой стороны синхронно — одна сущность);
- открытие → Конструктор в режиме просмотра (`/constructor?doc=…`) — коллега, живущий
  в Проводнике, даже не обязан знать, где раздел Конструктора;
- к зеркалу применимо всё «проводниковское»: теги (main/additional), статус (D/…),
  ревизия, отдел — документ участвует в документообороте наравне с чертежами;
- перенос зеркала в другую папку Проводником = смена папки из редактора (§3.2) —
  одно и то же действие с двух сторон;
- удаление зеркала в Проводнике = «в корзину Конструктора» с понятным подтверждением
  («это документ Конструктора — удалить его целиком?»); удалить «только из папки»
  нельзя — у документа всегда ровно одно расположение (ноль путаницы «где оригинал»);
- копирование зеркала → предлагается «Дублировать документ» (новый док + его зеркало).

Черновики зеркала не имеют (§3.3) — в Проводник попадает только именованное.

### 5.3. Синхронизация scope

Личный документ ⇢ зеркало в личной папке «Конструктор» владельца; общий ⇢ в общей.
Переключение scope автоматически перевозит зеркало (если оно лежало в пользовательской
подпапке — в корневую системную папку соответствующего раздела, с тостом «файл перенесён
в …»). Обратной рассинхронизации не существует по построению: личное зеркало в общей
папке невозможно — сервер валидирует пару (scope, папка) на каждом изменении.

### 5.4. Экспорт рядом с документом

Диалог экспорта (§I.10) получает умолчание: **та же папка, где зеркало**. Типовой
финал работы «сделал таблицу → PDF заказчику» = кнопка «Экспорт → PDF → Готово»,
файл лёг рядом с документом, ничего выбирать не пришлось. Экспортированный
PDF/XLSX/DOCX — обычный статический `FileNode` (снимок на момент экспорта), его
можно двигать/удалять как любой файл; связь «из какого документа» пишем в лог.

## 6. Изменения в моделях (дельта к части I/II)

```prisma
model ConstructorDoc {
  // … как в части I, плюс:
  scope        String    @default("SHARED")   // "SHARED" | "PERSONAL"
  ownerId      String?                        // = createdById; владелец личного
  named        Boolean   @default(false)      // имя дано человеком (гасит диалог закрытия)
  pinnedBy     String    @default("[]")       // JSON userId[] — закрепления
  deletedAt    DateTime?                      // корзина (30 дней)
  fileNodeId   String?                        // зеркало в Проводнике (null у черновиков)
}

model Folder {  system Boolean @default(false)  }        // защищённые папки
model FileNode { refId String? }                          // type="CONSTRUCTOR" → docId
```

«Последние мои» для ленты «Продолжить» — без новой таблицы: `AppSetting`
`constructor_recent`, userId=<я>, значение — массив `{docId, at}` (обновляется при
закрытии редактирования, хранит 10). Дёшево и в духе существующих настроек.

## 7. Как это дружит с уже спроектированным

- **Просмотр по умолчанию** идеально ложится на блокировки §II.10: read-only больше
  не «неудачный исход» (документ занят), а нормальное первое состояние; блокировка
  берётся только на явное «Редактировать».
- **Автосейв** (§II.7.2) делает диалог закрытия безопасным: «назвать файл» — это не
  «сохранить работу», работа уже в базе; диалог можно закрыть крестиком без потерь.
- **Stale-значок** на карточках Библиотеки — тот же `check-fresh` (§II.2.4), batch по
  всем документам списка одним запросом при входе в раздел.
- **Версии/конфликты/перехват** — без изменений; «Попросить» (§2) — новое, но это одно
  уведомление через существующий механизм Notifications.
- **ИИ-ассистент**: «открой мою вчерашнюю таблицу по клапанам» → поиск по
  Библиотеке + `constructor_recent` → карточка с кнопкой [Открыть] — ещё один вход
  за 0 кликов по разделам.

## 8. Крайние случаи (дополнение к реестру §II.15)

13. Автор общего документа уволен/деактивирован → документ остаётся общим; права
    «как автор» переходят к `constructor.manage`.
14. Владелец личного документа деактивирован → документ невидим никому; админская
    операция «передать личные документы пользователя X пользователю Y» (вместе с
    личными папками Проводника — общая задача, решается одинаково).
15. Переключение Общий→Личный, пока коллега редактирует → запрещено с подсказкой
    «сейчас редактирует Иванов» (не отбираем работу из-под рук); просмотрщикам —
    мягкое закрытие (§3.2).
16. Два пользователя одновременно жмут «Редактировать» → блокировку берёт первый
    (сервер атомарен), второй мгновенно видит «Редактирует …» — гонки нет.
17. Документ удалён в корзину, пока открыт у коллеги в просмотре → баннер «документ
    удалён автором», просмотр остаётся до закрытия (дочитать можно), редактирование
    недоступно.
18. Восстановление из корзины → зеркало возвращается в прежнюю папку; папки нет —
    в корневую системную, с тостом.
19. Проект удалён → каскад Prisma забирает документы и зеркала (FK уже так работают).
20. Импорт .xlsx как нового документа (§I.8) → тот же цикл: черновик → именование
    при закрытии (предложение имени = имя импортированного файла).

## 9. Контроль «минимума действий»: карта путей

Считаем клики от входа в раздел (вход = 1 клик по пункту меню, не считаем):

| Сценарий | Путь | Действий |
|---|---|---|
| Продолжить вчерашнюю работу | «Продолжить» → карточка | **1** |
| Посмотреть общий документ коллеги | карточка в «Общих» | **1** |
| …и продолжить его редактировать | + `E` / двойной клик по ячейке | **2** |
| Новый пустой документ и работать | `+ Новый` | **1** |
| Собрать таблицу по данным | «Собрать данные» → 5 шагов мастера → вставка | 7 |
| Закончить и назвать | `← Закрыть` → Enter (имя предложено) | **2** |
| Закончить черновиком | `← Закрыть` → «Оставить черновиком» | 2 |
| Сделать документ личным | селект в шапке → «Личный» | 2 |
| Экспорт PDF рядом с документом | «Экспорт» → PDF → «Готово» | 3 |
| Переложить в папку заказчика | 📁 в шапке → папка | 2 |
| Отдать документ просящему коллеге | закрыть редактор (у него — уведомление с [Открыть]) | 1 |
| Открыть таблицу из Проводника | клик по файлу-зеркалу | 1 |

Ни один частый сценарий не дороже 3 действий; единственный «длинный» — мастер, и его
длина оплачена содержанием (пять осмысленных шагов), а повторные запуски сокращаются
пресетами (§I.5) до: «Собрать данные» → пресет → «Вставить» = 3.
