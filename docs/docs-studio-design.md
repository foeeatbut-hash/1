# Студия документов — Конструктор 2.0: Таблица + Документ + Заметка

Статус: проект (продумывание). Реализация — после подтверждения.

Цель: один раздел, в котором живут **все рабочие документы** программы —
эксель-таблицы, ворд-документы и заметки. Файлы из Проводника (xlsx, docx,
txt и др.) открываются в этом же разделе: сначала просмотр, по желанию —
редактирование. Максимальная близость к настоящим Excel/Word, но с
функциями программы: живые данные проекта, статусы, версии, совместная
работа, печать.

---

## I. Ключевое технологическое решение

**Word-движок уже установлен.** Пакет `@univerjs/presets@0.25.1`, на котором
работает наш Эксель (preset-sheets-core), содержит и документные пресеты:

| Пресет | Что даёт |
|---|---|
| `preset-docs-core` | полноценный Word-редактор: страницы А4, абзацы, стили, шрифты, списки, таблицы, колонтитулы |
| `preset-docs-drawing` | картинки в документе (вставка, обтекание) |
| `preset-docs-hyper-link` | гиперссылки |
| `preset-docs-thread-comment` | комментарии к фрагментам (рецензирование) |

Следствия:
- **Ноль новых зависимостей** — Ворд получаем из уже лежащей библиотеки.
- **Единый вид** — панели, шрифты, поведение Ворда и Экселя одинаковы,
  раздел выглядит цельно, а не как три разные программы.
- **Единая коллаборация** — та же схема presence/патчей, что уже работает
  в таблицах (курсоры, подсветка выделения, мгновенные изменения).
- **Единое хранение** — snapshot Univer в ConstructorDoc, как у таблиц.
- Страничная вёрстка «как в Ворде» (лист А4 на сером фоне) — из коробки.

Блокнот: сейчас это собственный редактор на устаревшем `execCommand`
(RichTextEditor.tsx). Решение — **заметка = лёгкий документ** на том же
Univer Docs, но в режиме «без страниц» (сплошное полотно), с быстрыми
фишками заметок. Старый редактор остаётся только для чтения старых заметок
до их миграции (открыл старую → конвертация HTML → Univer, прозрачно).

## II. Архитектура раздела

### Один раздел — «Конструктор» (вкладки внутри)

```
┌──────────────────────────────────────────────────────────────┐
│  Конструктор                                [ + Создать ▾ ]  │
│  ┌────────┬─────────┬─────────┬──────────┐                   │
│  │ Все    │ Таблицы │Документы│ Заметки  │   [поиск…]        │
│  └────────┴─────────┴─────────┴──────────┘                   │
│  Недавние ─────────────────────────────────────────────      │
│  [͏͏▦ Ведомость П1] [📄 ПЗ раздел ОВ] [🗒 Звонок заказчику]   │
│  Общие ────────────────────────────────────────────────      │
│  ...карточки с типом, автором, статусом, датой...            │
│  Мои личные ───────────────────────────────────────────      │
└──────────────────────────────────────────────────────────────┘
```

- Кнопка **«+ Создать»**: Таблица · Документ · Заметка (+ «из шаблона»).
- Библиотека одна: карточки трёх типов различаются иконкой и цветом
  (таблица — изумруд, документ — синий, заметка — янтарь).
- Фильтры: тип, общие/личные, автор, статус, проект. Сортировка: недавние.
- Раздел «Блокнот» в меню становится ярлыком на вкладку «Заметки»
  (старый маршрут /notes редиректит — привычка пользователей не ломается).

### Данные

`ConstructorDoc` расширяется полем `kind`: `SHEET | DOC | NOTE`
(миграция: существующим — SHEET). Всё остальное — общее и уже есть:
scope (общий/личный), owner, folder в Проводнике, versions, snapshot.

Заметки из старой таблицы `UserNote` мигрируются лениво: при первом
открытии старая HTML-заметка конвертируется в ConstructorDoc(kind=NOTE),
оригинал помечается перенесённым. Список заметок объединённый до полного
переезда.

### Роутинг

- `/constructor` — библиотека (вкладки).
- `/constructor/doc/:id` — редактор (сам понимает kind и грузит нужный пресет).
- Ленивые чанки: preset-sheets-core и preset-docs-core грузятся отдельно
  и только при открытии соответствующего типа (страница списка лёгкая).

## III. Ворд-редактор (kind=DOC)

### Возможности (из preset-docs-core + наши надстройки)

**Как в Word:**
- страницы А4/А3, поля, ориентация (портрет/альбом);
- шрифты, размер, жирный/курсив/подчёркнутый/зачёркнутый, цвет,
  выделение цветом, верхний/нижний индекс;
- абзац: выравнивание, отступы, межстрочный интервал;
- стили заголовков (Заголовок 1–3, обычный) — по ним строится оглавление;
- маркированные и нумерованные списки, многоуровневые;
- таблицы (вставка, строки/столбцы, объединение, границы);
- картинки (drawing-пресет): вставка из файла и из буфера (скриншоты!);
- гиперссылки; поиск по документу; отмена/повтор;
- колонтитулы (шапка с шифром проекта — см. «умные поля»).

**Наши надстройки (функционал программы):**
1. **Умные поля** — вставка живых значений: `{Проект.Название}`,
   `{Проект.Код}`, `{Тег AHU-01 → Марка}`, `{Оборудование У-1 → Расход}`,
   `{Дата}`, `{Автор}`. Через тот же мастер «Данные», что в таблицах.
   Поле подсвечено; если в базе значение изменилось — плашка «данные
   обновились → [обновить]» (как в таблицах).
2. **Умные блоки**: титульный лист по ГОСТ-образцу, лист согласования
   (таблица подписей из сотрудников), спецификация (таблица из выбранного
   оборудования прямо в текст), штамп с реквизитами проекта.
3. **Вставка живой таблицы из эксель-документа** (фаза 2): фрагмент
   таблицы Конструктора как таблица в тексте, со ссылкой на источник.
4. **Комментарии** (thread-comment): выделил фрагмент → комментарий →
   коллега видит и отвечает. Основа согласования пояснительных записок.

### Импорт / экспорт DOCX

- **Импорт**: docx из Проводника → конвертация в Univer-документ.
  Путь: mammoth (docx→HTML на сервере) → HTML→Univer-снапшот.
  Переносятся: текст, заголовки, жирность/курсив, списки, таблицы,
  картинки. Сложная вёрстка (колонки, надписи) — упрощается, об этом
  честно предупреждаем при импорте («документ будет упрощён»).
- **Экспорт**: Univer→HTML→docx (библиотека docx, уже в планах зависимостей)
  + печать/PDF через существующий печатный маршрут (как у таблиц).
- Родной формат хранения — snapshot (без потерь); docx — обмен с внешним миром.

## IV. Эксель (kind=SHEET) — доводка

Уже есть: Univer-таблицы, формулы + функции программы, мастер данных,
умные блоки, шаблоны, версии, коллаборация, печать/PDF, зеркало в Проводник.

Добрать в рамках студии:
1. **Открытие xlsx из Проводника** в этом же редакторе (сейчас xlsx только
   парсится на импорт оборудования): xlsx → SheetJS → Univer-снапшот →
   документ типа SHEET «из файла» (просмотр → «Редактировать копию»).
2. **csv** — так же (одна таблица).
3. Экспорт в настоящий .xlsx (SheetJS из снапшота) — кнопкой.

## V. Блокнот 2.0 (kind=NOTE)

Заметка = документ без страниц: сплошное полотно (как OneNote/Notion),
тот же движок, панель сокращена до главного.

Улучшения против старого блокнота:
- нормальные таблицы и картинки (старый execCommand это делал криво);
- **чек-листы** (список задач с галочками) — самый частый кейс заметок;
- упоминание тега `#AHU-01` — становится ссылкой на карточку тега;
- быстрые цвета-«маркеры» и крупные заголовки одним кликом;
- сохраняются возможности старого блокнота: закрепить, дублировать,
  экспорт TXT, **стикер поверх окон** (открепление остаётся, стикер
  рендерит облегчённый просмотр);
- заметки по умолчанию **личные** (в отличие от таблиц/документов),
  переключаются в общие — тогда работает совместное редактирование
  (протокол заметок = протокол документов).

## VI. Проводник ↔ Студия: маршрутизация форматов

Двойной клик по файлу в Проводнике:

| Формат | Куда открывается | Режим |
|---|---|---|
| .xlsx, .xlsm, .csv | Студия → Таблица | просмотр → «Редактировать копию» |
| .docx, .doc, .rtf | Студия → Документ | просмотр → «Редактировать копию» |
| .txt, .md | Студия → Заметка/текст | сразу можно редактировать (просто) |
| документы Конструктора (зеркала) | Студия → родной редактор | родное редактирование |
| .pdf, картинки | как сейчас — предпросмотр Проводника | просмотр |
| прочее | как сейчас — скачивание/инфо | — |

Принципы:
- **«Редактировать копию»**: внешний файл — источник; редактирование
  создаёт документ Конструктора «на основе файла» (связь хранится:
  «создан из Ведомость.xlsx v2»). Исходник в Проводнике не портим —
  это регистр выданной документации. Обратно можно «Экспортировать в
  файл» (перезаписать новой ревизией со статусом, B1-логика статусов
  применяется).
- В контекстном меню Проводника: «Открыть в Конструкторе», «Создать
  здесь: Таблицу / Документ / Заметку» (создание сразу в текущей папке).
- В карточке файла (инспектор) — кнопка «Открыть в Конструкторе».

## VII. Общие механики (одинаковые для трёх типов)

1. **Просмотр по умолчанию, редактирование по кнопке** — как уже принято
   в Конструкторе. Заметки — сразу редактирование (это черновики).
2. **Статусы документооборота** (Черновик/На проверке/Согласован/Выдан) —
   те же, что в Проводнике, у любого документа студии; смена статуса из
   шапки редактора; «Выдан» блокирует редактирование (только новая версия).
3. **Версии** — единая панель (уже есть у таблиц): снимок при закрытии
   и вручную; просмотр/восстановление; у документов дифф «что изменилось»
   (фаза 2).
4. **Коллаборация** — presence + патчи для всех трёх типов; чат-«поделиться»
   ссылкой на документ (как уже умеем делиться экранами).
5. **Шаблоны** — единая галерея: шаблоны таблиц (есть) + шаблоны документов
   (титульник, ПЗ, письмо, протокол совещания, акт) + шаблоны заметок
   (чек-лист выезда, заметка звонка). Админ может сохранять свои шаблоны
   для всех.
6. **Печать/PDF** — единая кнопка; у документов постранично из коробки.
7. **Экспорт**: Таблица → xlsx/pdf; Документ → docx/pdf; Заметка → txt/pdf.

## VIII. Стороны использования (сценарии)

**Инженер:** пояснительная записка с живыми параметрами установок; ведомость
в таблице; заметка-чек-лист на монтаж; открыл присланный заказчиком docx —
прочитал, сделал редактируемую копию, вставил свои данные, выдал.

**Руководитель/ГИП:** лист согласования с подписями; комментарии в ПЗ
вместо правок «поверх»; статус «Согласован» одним нажатием; видит, кто
сейчас в документе.

**Менеджер по закупкам:** письмо поставщику из шаблона с автоподстановкой
позиций из Менеджмента; заметки по звонкам с упоминанием тегов.

**Секретарь/документооборот:** титульники и штампы по ГОСТ из умных блоков;
реестр выданного — статусы в Проводнике, экспортированные файлы рядом.

**Все вместе:** протокол совещания пишут одновременно трое; заметка
«вопросы к заказчику» общая — каждый дописывает; таблица выдач живёт
онлайн, а её печатная форма — документ со ссылкой на неё.

**Крайние случаи (продуманы):**
- большой docx (200 страниц) — импорт с прогрессом, при сбое честная
  ошибка «файл слишком сложный, открыт как просмотр»;
- битый/зашифрованный файл — сообщение, не падение;
- txt в другой кодировке — уже решено (TextDecoder), тот же путь;
- два человека переименовывают документ — last-write + событие;
- документ «Выдан» — редактирование закрыто, но «создать новую версию»;
- офлайн (встроенный сервер) — всё работает так же, коллаборация в
  пределах машины.

## IX. Риски и честные ограничения

1. Univer Docs моложе Univer Sheets: часть тонкостей Word (сноски,
   сложное обтекание, рецензирование-treck-changes) недоступна. Для
   инженерного документооборота хватает; для юридических договоров —
   останется настоящий Word (файл живёт в Проводнике).
2. Конвертация docx туда-обратно — с упрощениями; всегда предупреждаем
   и не трогаем исходник.
3. Вес бандла: doc-пресет — отдельный ленивый чанк, на старте не грузится.
4. Миграция заметок — ленивая и обратимая (оригинал не удаляется).

## X. Фазы внедрения

| Фаза | Состав | Итог для пользователя |
|---|---|---|
| 1 | Вкладки в Конструкторе, kind в модели, Word-редактор (docs-core+drawing+link), создание/сохранение/версии/статусы, печать | «Создать документ» — настоящий Ворд внутри программы |
| 2 | Заметки на новом движке + миграция старых, чек-листы, стикер | Блокнот переезжает в студию, старое не теряется |
| 3 | Проводник: открытие docx/txt/xlsx в студии, «Редактировать копию», экспорт docx/xlsx | Любой офисный файл открывается в программе |
| 4 | Умные поля и умные блоки в Ворде, шаблоны документов, комментарии | Документы наполняются живыми данными |
| 5 | Коллаборация в документах/заметках, дифф версий | Совместные ПЗ и протоколы |

Каждая фаза — самостоятельно полезна, собирается и проверяется отдельно.
