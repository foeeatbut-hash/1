# ИИ-чат: проект «помощника, с которым легко пользоваться программой»

Только проектирование, без применения. Цель — чтобы через чат можно было и **узнать**,
и **найти**, и **сделать**, не зная, где что лежит и как называется. Остаётся локальным
и офлайн (это преимущество, не меняем).

Главный принцип: **на каждый ответ — действие**. Не «зайдите в раздел…», а кнопка,
которая уже туда ведёт или сразу делает.

---

## 1. Как человек на самом деле обращается — четыре роли, четыре режима

Чат должен одинаково хорошо отвечать четырём типам вопросов, не заставляя выбирать режим:

| Что спрашивают | Пример | Что выдаёт |
|---|---|---|
| **Найти** (данные) | «покажи вытяжные вентиляторы», «где тег 3700-K02» | таблица/карточка + кнопки действий |
| **Сделать** (навигация/команда) | «создай заметку», «открой менеджмент», «найди дубли» | выполняет + подтверждает |
| **Как** (обучение) | «как связать теги», «как импортировать бланк» | демонстрация с подсветкой |
| **Что/почему** (справка) | «что такое спецификация», «почему конфликт ревизии» | короткий ответ + «показать» |

Определение намерения — по глаголу и сущности (уже частично есть). Добавить: если в
запросе распознан **код тега/марка/система** — это почти всегда «найти», даже без глагола
(«3700-K02-HV-209» → сразу карточка позиции).

---

## 2. Понимать «как говорят», а не «как в справочнике»

Сейчас `tokensFrom` режет 2 буквы — теряются склонения и опечатки. Нужно:

1. **Лёгкий стеммер** русского (отрезание окончаний по таблице), чтобы «вентиляторов»,
   «вентилятору», «вентилятор» сводились к одной основе. Без внешних библиотек.
2. **Допуск 1 опечатки** (расстояние Левенштейна на основах): «спецификацыя»,
   «менеджмнт», «оборудвание» находятся.
3. **Синонимы бытовой речи** → термины: «коробки»→AHU/установки, «движок»→двигатель,
   «сколько всего»→счётчики, «просрочка»→этап закупки, «что не так»→проблемы.
4. **Числа и коды** распознавать как есть: `3700-K02`, `ВР-80`, `П1`.

Порог фоллбэка поднять по осмысленности: если совпадение слабое — не выдавать мусорный
ответ, а честно предложить 2–3 темы (см. §7).

---

## 3. Расширить, о чём вообще можно спросить (сейчас только теги+компоненты)

Добавить домены, которые закрывают ежедневные вопросы. Всё — из уже существующих данных:

- **Закупки/Менеджмент**: «на каком этапе клапаны», «что ещё не заказано»,
  «покажи просроченное», «что закупили за неделю». → таблица + кнопка «Открыть в Менеджменте».
- **Дубли**: «покажи дубли», «сколько дублей» → список + кнопка «Найти дубли на холсте»
  (та самая панель с прокруткой, что уже сделали).
- **Проблемы/актуальность**: «что требует внимания», «критичные позиции»,
  «конфликты ревизий оборудования» → список с переходом к каждому.
- **Проект**: «готовность проекта», «сводка», «сколько всего» → карточка показателей.
- **Заметки**: «найди заметку про калорифер» → поиск по содержимому, открыть.
- **История/логи**: «что изменилось сегодня», «кто менял 3700-K02», «последние действия».
- **Оборудование**: «характеристики 3700-C03-BL-001», «расход этого вентилятора» → карточка.

Каждый домен = маленький распознаватель по образцу текущего блока «данные».

---

## 4. Ответ = карточка + действия, а не стена текста

Сейчас максимум — таблица + «выгрузить в Excel/Word». Добавить контекстные кнопки под
ответом, привязанные к найденному:

- Нашли тег/позицию → **«Открыть на холсте»**, **«Показать в Менеджменте»**,
  **«Найти дубли»**, **«Поделиться в чате»**, **«В блокнот»**.
- Нашли оборудование → **«Открыть карточку»**, **«Сравнить ревизии»**.
- Счётчик/сводка → **«Открыть раздел»**.
- Любой список → **«Выгрузить в Excel/Word»** (есть) + **«Открыть отфильтрованным»**
  (перейти в раздел с уже применённым фильтром).

Это превращает чат из справочной в пульт управления: спросил — и одной кнопкой уже там.

---

## 5. Навигация и команды-действия

Чат должен уметь выполнять, а не только рассказывать:

- «открой менеджмент / теги / оборудование» → переход (у нас есть `navigateFn`).
- «создай заметку [про X]» → создаёт и открывает заметку.
- «найди дубли [кода]» → открывает панель дублей.
- «выбери проект X» → переключает активный проект.
- «покажи 3700-K02 на холсте» → переход + центрирование карточки (есть `centerOnTag`).
- «сверни/разверни помощника», «тёмная тема» → мелкие UI-команды.

Каждая команда подтверждается одной строкой («Готово: открыл Менеджмент»), без воды.

---

## 6. Контекст диалога — чтобы не повторять

Хранить последний предмет разговора (тип + фильтр + найденный список):

- «покажи вентиляторы вытяжки» → список; затем «а сколько их?» → берёт из контекста;
  «выгрузи» → экспортит тот же список; «первый на холсте» → открывает первую позицию.
- «этот тег» / «эта позиция» / «они» — разрешаются в последний объект.

Достаточно одного слота «последний результат» в сторе — недорого, а удобство большое.

---

## 7. Проактивность и мягкая посадка новичка

Чтобы «легко пользоваться» с первого дня:

- **Первый запуск / пустой проект**: помощник сам предлагает «С чего начать» —
  3 шага (выбрать проект → добавить теги/импортировать → связать). Кнопками.
- **Контекстные подсказки раздела** уже есть — усилить: подсказки зависят от состояния
  («в проекте нет тегов» → «Импортировать из таблицы»; «есть дубли» → «Показать дубли»).
- **Честное «не знаю»**: вместо общего фоллбэка — «Не понял. Возможно, вы про: [Дубли]
  [Этапы закупки] [Как импортировать]» — три кликабельные темы.
- **Замечает проблемы**: при заходе в проект с критичными позициями/дублями — ненавязчивая
  строка «В проекте 4 дубля и 2 критичные позиции — показать?».

---

## 8. Мелочи, которые делают чат приятным

- **Быстрые чипы над полем ввода** — 3–4 самых частых запроса для текущего раздела
  (уже частично есть в подсказках; довести до «одного клика»).
- **История ввода** стрелкой вверх (как в терминале).
- **Ctrl+K / горячая клавиша** — открыть помощника и сразу фокус в поле (для «спросить на лету»).
- **Копка «копировать ответ»** и «повторить запрос».
- **Кнопка озвучки не нужна** (шумный цех — бесполезно), не добавляем.
- **Ссылки на позиции внутри ответа кликабельны** — код тега в тексте ведёт на холст.

---

## 9. Что осознанно НЕ делаем

- Не подключаем внешние LLM/интернет — теряется офлайн и приватность данных.
- Не превращаем в «болталку» — помощник по программе, а не собеседник.
- Не плодим режимы (сейчас есть тумблер «Демонстрация») — намерение определяется само,
  тумблер можно оставить как «всегда показывать пошагово» для обучения новичка.

---

## 10. Порядок реализации (когда скажешь)

1. Понимание речи: стеммер + опечатки + синонимы (сразу поднимает попадаемость).
2. Action-кнопки под ответами + навигационные команды (превращает в пульт).
3. Расширение доменов: дубли, этапы закупки, проблемы, сводка проекта.
4. Контекст диалога («а сколько их / этот тег»).
5. Проактивность: мягкая посадка, честное «не знаю», подсказки по состоянию.

Каждый пункт — самостоятельный и проверяемый; можно останавливаться и смотреть результат.
