# PDM System (MAX)

Корпоративная инженерная система управления данными (PDM/EDMS): проекты,
документооборот, реестр тегов и оборудования, файловый проводник, рабочий
чат и встроенный локальный ИИ-ассистент.

Приложение работает **полностью локально** (Electron + встроенный Express +
SQLite через Prisma). Внешние ИИ-сервисы не используются.

## Запуск в разработке

```bash
npm install
npm run dev      # поднимает сервер + Vite на http://localhost:3000
```

## Сборка portable-приложения (Windows)

```bash
npm run dist     # собирает фронтенд, сервер, Electron и portable .exe
```

Готовый exe появится в папке `release/`.

## Данные

Локальная база `database.sqlite` создаётся в `%APPDATA%/pdm-app` при первом
запуске. Путь можно изменить на экране входа («Выбрать файл БД…») или в
профиле пользователя.

## Сервер компании (общая работа по сети)

Приложение работает в двух вариантах подключения (переключается на экране
входа, кнопка под формой):

- **Встроенный сервер (эта машина)** — по умолчанию: полностью автономная
  работа, как описано выше;
- **Сервер компании** — все данные, чат и события в реальном времени общие
  для сотрудников.

Развёртывание сервера на офисной машине:

```bash
npm run build                      # собирает фронтенд и dist/server.cjs
node dist/server.cjs               # поднимает сервер на порту 3000
```

БД сервера настраивается через `%APPDATA%/pdm-app/config.json`
(`current_db_type`: `LOCAL` для SQLite или `REMOTE` + `database_url` для
PostgreSQL — для многопользовательской работы рекомендуется PostgreSQL).
Сервер раздаёт и сам интерфейс: программа открывается в браузере по
`http://адрес-сервера:3000` без установки. Клиенты в Electron указывают этот
же адрес на экране входа («Проверить» → «Подключиться»); встроенный сервер
у них при этом не запускается.
